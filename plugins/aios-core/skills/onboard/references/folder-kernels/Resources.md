# Resources (the reusable library)

> [!important] Language: these instructions are in English; the user is Hebrew-speaking. Reason and run the steps in English, but write everything the user sees in Hebrew: chat replies, reports, vault notes, tables, status lines, task text. Keep commands, file paths, field names, dates, and numbers as they are.

This is where everything you want to reuse lives: prompts, templates, frameworks, tool guides, and inspiration material. Instead of building from scratch every time, you pull from here. When something works well and is worth keeping, its home is here. Whatever is written here for you is written in Hebrew.

## Structure

| Subfolder | What goes in |
| --- | --- |
| `frameworks/` | Reusable working methods and thinking frameworks |
| `templates/` | Content templates (Markdown, HTML) |
| `prompts/` | Saved prompts for recurring tasks |
| `tools/` | Documentation on tools and platforms you use |
| `swipe/` | Inspiration material and good examples you have seen |
| `archive/` | Material that was replaced or went stale |

(Folders are created on demand, not up front.)

## Rules

- When a user shares reusable content (a prompt, a framework, a template), save it here with a descriptive file name.
- Keep light nesting: use the existing subfolders, create a new one only when there is a clear category.
- Do not duplicate content that already lives in a project folder. Link to it with a `[[link]]` instead of copying.
- Use `[[wikilinks]]` to reference a resource from project notes or the journal.
